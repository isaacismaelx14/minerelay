# Security Policy

## Supported versions

Only the latest commit on `main` is considered supported for security fixes.

## Reporting a vulnerability

Do not open public issues for security vulnerabilities.

Use GitHub private vulnerability reporting for this repository. If that option
is unavailable, contact maintainers privately on GitHub with:

- Vulnerability description
- Affected component and version/commit
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation (optional)

## Response targets

- Initial acknowledgment: within 5 business days
- Triage decision: within 10 business days
- Fix timeline: depends on severity and release risk

## Disclosure

Please follow coordinated disclosure. Public disclosure should wait until a fix
or mitigation is available.
